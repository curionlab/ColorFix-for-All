# ColorFix for All

> 色をなおす、デザインを守る。機密文書対応のPDFアクセシビリティツール。

ColorFix for All は、AIのハルシネーション（非決定的な推論）に頼らず、**「数理計算」**によって文書のカラーアクセシビリティを評価・修正するツールです。WCAG 2.2や、色覚多様性・ロービジョンに配慮した ISO 24505-2:2025 規格を参照し、ブランドカラーの「色相」を保持したまま、最低限の「明度・彩度（CIE Lab色空間）」の調整で識別性を確保できる代替カラーを提案します。

元々はHTML解析ベースのツールとして開発されましたが、PDF解析システムへと完全にピボットしました。

## 🚀 Core Features (主要機能)

1. **セキュアな PDF Parsing (Client-side)**
   `pdf.js` を利用して、ブラウザ内で完結するPDFのテキスト抽出と背景色サンプリングを実現。サーバーに機密ファイルを送信することなく、完全オフラインで解析が行われるため安全です。

2. **Constrained Optimization in OKLCH (OKLCH色空間の制約付き最適化)**
   色操作に不向きな RGB や HSL ではなく、最新の知覚均一色空間である **OKLCH** を採用。元のブランドカラーの「Hue（色相）」を完全にロック（固定）した状態で、通常視および CVD シミュレーション環境下のコントラスト制約を満たす最小の変化量を探索します。
   - **OKLCH の採用**: CIELAB よりも色相の維持性能が高く、人間の知覚に極めて近い調整が可能です。
   - **多段階最適化**: 単純なグリッド探索ではなく、Lightness（明度）の二分探索と Chroma（彩度）の調整を組み合わせることで、ブランドイメージへの影響（Delta E）が最も少ない最適なペア（文字色・背景色）を算出します。

3. **WCAG Compliance (Webアクセシビリティ標準)**
   WCAG 2.2 に基づく相対輝度（Luminance）を用いたコントラスト比の計算を提供。Webにおける標準的なアクセシビリティ要件（AA/AAA）を満たすかどうかを自動判定します。

4. **ISO 24505-1 & 24505-2 Support (色覚多様性・ロービジョン対応)**
   国内（産総研）で研究・公開されている最新の国際規格の考え方に準拠。
   - **P型・D型（赤緑色覚特性）、T型（青黄色覚特性）**の方の視知覚（1D・2D色空間への退行）をシミュレーションし、同化して見えてしまう色同士の組み合わせを回避します。
   - **ロービジョン**（弱視や高齢者）の方がオブジェクトの輪郭を素早く区別できるよう、明度差（L*値）を十分に広げる指標を用いています。

5. **Color Map Export (推奨色マップのエクスポート)**
   解析結果から得られた「修正すべき色」と「推奨色」の対応表を JSON および CSV 形式で出力。デザイナーが Figma やデザインツールに値を反映させる際のワークフローを支援します。


## 🏗️ Architecture (アーキテクチャ)

本リポジトリは Monorepo 構成を採用しています。以前使用されていた `packages/core`, `packages/fixtures`, `packages/schemas` などのレガシーアセットは、すべて `old_version/` ディレクトリに退避されています。

- **`apps/web/`** (Astro + React + Tailwind CSS)
  - ユーザーインターフェースおよびPDF Dropzone機能を提供。
  - Astro による静的および軽量なランディングページの構築と、React によるインタラクティブな `WorkspaceLayout` で構成されています。
  - レガシーLP（`old_version/colorfix_for_all_lp.html`）の「ダークグリーン・ブランド」を踏襲したセキュアなUI設計へ移行しました。

- **`packages/color-engine/`** (Pure TypeScript)
  - ColorFix の心臓部となる、純粋な色計算アルゴリズムを提供するライブラリ。
  - WCAG コントラスト計算、Delta E ベースの色差計算、ISO 24505 CVD/ロービジョン シミュレーション関数群を含みます。
  - 状態を持たない関数の集まりであるため、外部環境（Node, ブラウザ, サーバーレス等）に依存せず deterministic（決定的）に動作します。

## 💡 使用方法

開発環境をセットアップし、ツールを起動するには以下の手順を実行します：

```bash
# パッケージのインストール
npm install

# 開発サーバーの起動
npm run dev
```

ブラウザで `http://localhost:4321` にアクセスし、「PDFで試す」ボタンをクリックしてください。表示されたワークスペースでPDFファイルをドラッグ＆ドロップすると解析が始まります。

> **※注意**: 本リポジトリの検証ディレクトリにある `test.pdf` などを利用して動作確認を行うことができます。

## 📐 計算基準と理論的背景 (Calculation Standards & Theoretical Basis)

当アプリは、正確な色評価を目指していますが、以下の理論的背景と制約に基づいています。

- **知覚的均一性 (OKLCH vs. CIELAB)**: 本ツールでは、色の探索空間として **OKLCH** を採用しています。CIELAB（L*a*b*）は青色の変化で色相がずれやすい欠点（Blue-shift）がありますが、OKLCH はこれを克服しており、アクセシビリティ対応のための明度調整を行っても「元の色の印象（色味）」が極めて正確に維持されます。
- **色差の評価 (CIEDE2000 / ΔE₀₀)**: 提案色と元のブランドカラーの距離（元の色味をどれだけ維持できているか）を測定するために、人間の視覚に最も近い均一な色差計算式 **CIEDE2000** (Sharma et al. 2005) を採用しています。
- **制約付き最適化 (Constrained Optimization)**: 
  - **Objective (目的関数)**: 元の色との知覚色差 $\Delta E_{00}$ の最小化。
  - **Constraints (制約条件)**: 通常視および CVD 3種（P/D/T型）のすべてにおいて、ターゲットとするコントラスト比（例: 4.5:1）を満たすこと。
- **用途別の判定指標 (Semantic vs Text)**:
  - **Semantic (凡例・カテゴリ等)**: 「色そのもの」での識別を重視。CVDシミュレーション（P/D/T型）後の **最小知覚色差 ΔE₀₀ ≥ 18** および **明度差 ΔL* ≥ 20** を基準とします。
  - **Text (文字・アイコン)**: 上記の識別性に加え、通常視およびCVDシミュレーション後の **コントラスト比 ≥ 4.5:1 (WCAG AA)** を可読性の補助指標として適用します。
- **CVD（色覚特性）シミュレーション**: P型・D型・T型の見え方の推定には、各特性特有の混同線に基づく行列変換（Viénot 1999 / Brettel 1997系）を採用し、シミュレートされた色空間上で再度色差・コントラスト計算を行っています。
